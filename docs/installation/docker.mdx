This guide walks you through installing and running VectorAI DB as a Docker container on your local machine.

## Prerequisites

Before installing, ensure you have the following minimum system requirements.
- At least 8 GB RAM (16 GB+ recommended)
- 10 GB disk space (100 GB+ recommended)

<Warning>
**Docker required**: VectorAI DB runs as a Docker container. Make sure Docker is installed and running on your system before proceeding. [Get Docker](https://docs.docker.com/get-docker/)
</Warning>

<Tip>
For best performance, use the recommended configuration.
</Tip>

## Pull and run VectorAI DB

Choose one of the following options depending on your needs.

### Option 1: Quickstart (single command)

Pull and run the VectorAI DB Docker container.

```bash
docker pull actian/vectorai:latest
docker run -p 6574:6574 -p 6575:6575 actian/vectorai:latest
```

For this and other options, the gRPC server is available at `localhost:6574`. The Web UI is available at `localhost:6575`.

### Option 2: With data persistence

To persist your data between container restarts, mount a local directory as a volume.

```bash
docker run -p 6574:6574 -p 6575:6575 \
  -v $(pwd)/data:/var/lib/actian-vectorai \
  actian/vectorai:latest
```

### Option 3: Using Docker Compose

Create a `docker-compose.yml` file:

```yaml
services:
  vectorai:
    image: actian/vectorai:latest
    ports:
      - "6574:6574"
      - "6575:6575"
    volumes:
      - ./data:/var/lib/actian-vectorai
    restart: unless-stopped
```

Run the following commands to manage the service.

Start the service.

```bash
docker-compose up -d
```

Stop the service.

```bash
docker-compose down
```

## Verify installation

<Warning>
**Python SDK Required**: To verify your installation, you need to have the VectorAI Python SDK installed. Install it using `pip install actian-vectorai` or follow the [SDK installation guide](/sdks/python/installation).
</Warning>

Test your installation by checking the server health:

```python
import asyncio
from actian_vectorai import AsyncVectorAIClient, VectorParams, Distance

async def main():
    async with AsyncVectorAIClient("localhost:6574") as client:
        info = await client.health_check()
        print(f"Connected to {info['title']} v{info['version']}")

asyncio.run(main())
```

## Troubleshooting

If you run into problems, check the table below for common issues and solutions.

| Issue                | Solution                                                        |
|----------------------|-----------------------------------------------------------------|
| Connection failed    | Verify the Docker container is running with `docker ps`. |
| Port already in use  | Stop services using port 6574 or use a different port: `-p 6576:6574` |
| Permission denied    | Run Docker commands with appropriate permissions or add your user to the docker group. |
| Container exits      | Inspect the logs with `docker logs <container-id>`. |

### View Docker logs

If a container exits unexpectedly or fails to start, then inspect the logs to identify the cause.

```bash
# List running containers
docker ps

# View logs
docker logs <container-id>

# Follow logs in real-time
docker logs -f <container-id>
```
